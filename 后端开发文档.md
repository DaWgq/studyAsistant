# 智慧校园管理系统 - 后端开发文档

## 1. 项目概述

### 1.1 项目简介
智慧校园管理系统是一款面向教师的学生信息管理应用，支持微信小程序等多端运行。系统提供学生档案管理、考勤统计、通知待办、个人中心等核心功能。

### 1.2 技术栈
- **前端框架**: uni-app (Vue 3)
- **目标平台**: 微信小程序 (AppID: wxcc1baa57046d4fc7)
- **推荐后端**: Node.js + Express / Python + FastAPI / Java + Spring Boot

## 3. 数据模型设计

### 3.1 用户表 (users)

存储教师/管理员用户信息

```sql
CREATE TABLE users (
    id              VARCHAR(32) PRIMARY KEY,      -- 用户 ID
    username        VARCHAR(50) NOT NULL,         -- 用户名
    password_hash   VARCHAR(255) NOT NULL,        -- 密码哈希
    real_name       VARCHAR(50) NOT NULL,         -- 真实姓名 (如：李华)
    employee_id     VARCHAR(20) UNIQUE,           -- 工号 (如：T89201)
    role            VARCHAR(20) DEFAULT 'teacher',-- 角色：teacher/admin
    avatar_url      VARCHAR(255),                 -- 头像 URL
    phone           VARCHAR(20),                  -- 联系电话
    email           VARCHAR(100),                 -- 邮箱
    status          TINYINT DEFAULT 1,            -- 状态：1-正常 0-禁用
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 3.2 学生表 (students)

存储学生基本信息

```sql
CREATE TABLE students (
    id              VARCHAR(32) PRIMARY KEY,      -- 学生 ID
    student_no      VARCHAR(20) UNIQUE NOT NULL,  -- 学号 (如：2023001)
    name            VARCHAR(50) NOT NULL,         -- 姓名
    gender          VARCHAR(10),                  -- 性别
    avatar_url      VARCHAR(255),                 -- 头像 URL
    grade           VARCHAR(50) NOT NULL,         -- 班级 (如：高一 (1) 班)
    major           VARCHAR(50),                  -- 文理分科 (理科/文科)
    phone           VARCHAR(20),                  -- 联系电话
    address         VARCHAR(255),                 -- 家庭住址
    status          VARCHAR(20) DEFAULT '在校',   -- 状态：在校/请假/缺勤
    enrollment_year INT,                          -- 入学年份
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_grade (grade),
    INDEX idx_status (status),
    INDEX idx_student_no (student_no)
);
```

### 3.3 通知表 (notifications)

存储系统通知和待办事项

```sql
CREATE TABLE notifications (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    user_id         VARCHAR(32) NOT NULL,         -- 接收用户 ID
    title           VARCHAR(200) NOT NULL,        -- 通知标题
    content         TEXT,                         -- 通知内容
    type            VARCHAR(20) NOT NULL,         -- 类型：通知/待办
    priority        VARCHAR(20) DEFAULT 'normal', -- 优先级：high/normal/low
    is_read         TINYINT DEFAULT 0,            -- 是否已读：0-未读 1-已读
    related_type    VARCHAR(50),                  -- 关联类型 (如：student_leave)
    related_id      VARCHAR(32),                  -- 关联 ID
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_is_read (is_read),
    INDEX idx_created_at (created_at)
);
```

### 3.4 考勤表 (attendance)

存储学生考勤记录

```sql
CREATE TABLE attendance (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    student_id      VARCHAR(32) NOT NULL,         -- 学生 ID
    date            DATE NOT NULL,                -- 考勤日期
    status          VARCHAR(20) NOT NULL,         -- 状态：出勤/请假/缺勤
    remark          VARCHAR(255),                 -- 备注
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_student_date (student_id, date),
    INDEX idx_date (date)
);
```

### 3.5 成绩表 (scores)

存储学生成绩信息

```sql
CREATE TABLE scores (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    student_id      VARCHAR(32) NOT NULL,         -- 学生 ID
    subject         VARCHAR(50) NOT NULL,         -- 科目
    exam_type       VARCHAR(50),                  -- 考试类型 (期中/期末/月考)
    score           DECIMAL(5,2),                 -- 分数
    class_rank      INT,                          -- 班级排名
    term            VARCHAR(50),                  -- 学期 (如：2023-2024-1)
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_student_id (student_id),
    INDEX idx_term (term)
);
```

### 3.6 教师陪伴天数表 (teacher_days)

存储教师陪伴学生天数统计

```sql
CREATE TABLE teacher_days (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    user_id         VARCHAR(32) NOT NULL,         -- 教师 ID
    total_days      INT DEFAULT 0,                -- 总天数
    start_date      DATE,                         -- 开始日期
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user (user_id)
);
```

## 5. 错误码规范

### 5.1 通用错误码

| 错误码 | 说明 |
|-------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权/Token 过期 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 5.2 业务错误码

| 错误码 | 说明 |
|-------|------|
| 1001 | 用户名或密码错误 |
| 1002 | Token 无效 |
| 1003 | Token 已过期 |
| 2001 | 学生不存在 |
| 2002 | 学号已存在 |
| 3001 | 通知不存在 |

### 5.3 错误响应格式

```json
{
  "code": 1001,
  "message": "用户名或密码错误",
  "data": null
}
```

## 6. 安全规范

### 6.1 认证机制
- 采用 JWT (JSON Web Token) 进行身份认证
- Token 有效期建议设置为 7 天
- 支持 Refresh Token 机制

### 6.2 密码安全
- 密码使用 bcrypt 或 Argon2 进行哈希存储
- 传输过程使用 HTTPS 加密

### 6.3 接口安全
- 所有需要认证的接口必须在 Header 中携带 `Authorization: Bearer {token}`
- 实现请求频率限制 (Rate Limiting)
- 实现 SQL 注入防护

### 6.4 数据权限
- 教师只能访问所负责班级的学生数据
- 管理员可访问全部数据

### 10.2 前端请求封装建议

```javascript
// utils/request.js
const BASE_URL = 'https://api.yourschool.com'

function request(options) {
  return new Promise((resolve, reject) => {
    const token = uni.getStorageSync('token')
    
    uni.request({
      url: BASE_URL + options.url,
      method: options.method || 'GET',
      data: options.data || {},
      header: {
        'Content-Type': 'application/json',
        'Authorization': token ? `Bearer ${token}` : ''
      },
      success: (res) => {
        if (res.data.code === 200) {
          resolve(res.data.data)
        } else if (res.data.code === 401) {
          // Token 过期，跳转登录
          uni.navigateTo({ url: '/pages/login/login' })
          reject(res.data)
        } else {
          uni.showToast({ title: res.data.message, icon: 'none' })
          reject(res.data)
        }
      },
      fail: (err) => reject(err)
    })
  })
}

export default {
  get: (url, data) => request({ url, method: 'GET', data }),
  post: (url, data) => request({ url, method: 'POST', data }),
  put: (url, data) => request({ url, method: 'PUT', data }),
  delete: (url, data) => request({ url, method: 'DELETE', data })
}
```
